# Dumblr ‚Äî Claude Context

Personal creative journal/logbook. Static site hosted on Netlify. No framework, no SSR ‚Äî pure HTML/CSS/JS.

---

## Architecture

- **Entries**: `entries/YYYY-MM/MMDD-slug/index.html` ‚Äî each entry is a standalone HTML file
- **Metadata**: embedded in each entry as `<script type="application/json" id="entry-meta">` JSON
- **Manifest**: `manifest.json` is auto-generated by `scripts/build-manifest.js` ‚Äî run `npm run build` after adding/editing any entry
- **Collections**: root-level HTML pages (`movies.html`, `books.html`, etc.) powered by `assets/js/collection.js`
- **Main page**: `index.html` powered by `assets/js/site.js`
- **Styles**: `assets/css/site.css` ‚Äî all site chrome styles; entries can have their own inline `<style>`

---

## Critical Rules

- **Never invent content.** Only add information explicitly provided by the user ‚Äî no descriptions, tags, directors, genres, etc. unless stated. Leave fields empty (`""`) if not provided.
- Always run `npm run build` after creating or modifying entries.
- When editing a file, always read it first (Edit tool requires a prior read).
- **Back links**: every entry's `‚Üê back` link must point to its collection page, not `/`:
  - movie ‚Üí `/movies.html`, book ‚Üí `/books.html`, theater ‚Üí `/theater.html`
  - meal ‚Üí `/meals.html`, outfit ‚Üí `/outfits.html`, memory ‚Üí `/memories.html`
  - travel ‚Üí `/travel.html`, shelf ‚Üí `/shelf.html`, interactive ‚Üí `/interactive.html`
  - quote ‚Üí `/quotes.html`, note ‚Üí `/notes.html`

## Entry Types & Metadata Fields

### movie
```json
{
  "title": "",
  "date": "YYYY-MM-DD",
  "type": "movie",
  "description": "",
  "tags": [],
  "pinned": false,
  "director": "",
  "year": 0,
  "where": "",
  "with": "",
  "rating": 4.5,
  "poster": "https://image.tmdb.org/t/p/w300/POSTER_PATH.jpg"
}
```
- `rating`: 0‚Äì5, halves supported (e.g. 4.5)
- `poster`: TMDB CDN ‚Äî no API key needed for image URLs
- Uses **poster layout** (Letterboxd-style grid) ‚Äî config in `movies.html` has `"layout": "poster"`

## Photo Workflow

Photos for memories and meals are stored as **regular files** in the entry directory (not base64). This keeps `manifest.json` lean regardless of how many photo entries exist.

**Memory entries** ‚Äî two files per entry:
```bash
# Full size for entry page (photo.jpg):
sips -Z 900 -s format jpeg -s formatOptions 60 input.jpg --out photo.jpg

# Thumbnail for collection card (thumb.jpg):
sips -Z 400 -s format jpeg -s formatOptions 35 input.jpg --out thumb.jpg
```
- Save both in the entry directory (e.g. `entries/2026-02/0228-apres-ski/`)
- Metadata `"photo": "thumb.jpg"` ‚Äî just the filename, not a data URI
- Entry HTML: `<img src="photo.jpg">`

**Meal entries** ‚Äî one file per photo:
```bash
sips -Z 900 -s format jpeg -s formatOptions 60 input.jpg --out photo-1.jpg
# repeat for photo-2.jpg, photo-3.jpg etc.
```
- Save in the entry directory
- Entry HTML: `<img src="photo-1.jpg">`, `<img src="photo-2.jpg">` etc.
- No photo field in meal metadata (meals don't show photos in collection cards)

Target sizes after compression:
- Full / meal photo: ~100‚Äì150KB
- Memory thumbnail: ~15‚Äì25KB

**Exception**: If the user explicitly asks for base64 for a specific entry, embed as `data:image/jpeg;base64,...` in the HTML only ‚Äî never in metadata/manifest.

## Sidebar Nav Structure

All 12 pages share the same sidebar. When adding a new collection, update **all** of these files:
`index.html`, `movies.html`, `books.html`, `theater.html`, `meals.html`, `travel.html`, `shelf.html`, `interactive.html`, `outfits.html`, `quotes.html`, `notes.html`, `memories.html`

Sidebar layout ‚Äî three sections (section labels are not links):
```
log
  movies       ‚Üí movies.html
  books        ‚Üí books.html
  shows        ‚Üí theater.html  (display name "shows", file is theater.html)
  meals        ‚Üí meals.html

life
  notes        ‚Üí notes.html
  memories     ‚Üí memories.html
  travel       ‚Üí travel.html
  shelf        ‚Üí shelf.html
  outfits      ‚Üí outfits.html

for fun
  quotes       ‚Üí quotes.html
  apps         ‚Üí interactive.html  (display name "apps", file is interactive.html)
```

Each collection page sets `class="sidebar-link active"` on its own link.
index.html has no active link (home page).

## Build & Deploy

```bash
npm run build        # regenerates manifest.json from all entries
```

**Deploy workflow:**
- After making changes, run `npm run build` and commit ‚Äî but **do NOT push**.
- Only run `git push` when the user explicitly says "deploy".
- Netlify auto-deploys on push to `main`.

## Star Ratings

- Scale: 0‚Äì5 with halves (e.g. 4.5)
- If user gives a /10 score, divide by 2
- Rendered via `renderStars()` in `collection.js` using CSS half-star clip trick

## Design Tokens (site.css)

- Font: Inter (Google Fonts, weights 400/500/600/700)
- Background: `#f9f7f4` (light) / `#141414` (dark)
- Accent: `#3a5ccc` (light) / `#7fa8ff` (dark)
- Theme: `data-theme="auto|light|dark"` on `<html>`, persisted in localStorage
- Border radius: `--radius: 10px`
