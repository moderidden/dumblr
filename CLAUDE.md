# Dumblr ŌĆö Claude Context

Personal creative journal/logbook. Static site hosted on Netlify. No framework, no SSR ŌĆö pure HTML/CSS/JS.

---

## Architecture

- **Entries**: `entries/YYYY-MM/MMDD-slug/index.html` ŌĆö each entry is a standalone HTML file
- **Metadata**: embedded in each entry as `<script type="application/json" id="entry-meta">` JSON
- **Manifest**: `manifest.json` is auto-generated by `scripts/build-manifest.js` ŌĆö run `npm run build` after adding/editing any entry
- **Collections**: root-level HTML pages (`movies.html`, `books.html`, etc.) powered by `assets/js/collection.js`
- **Main page**: `index.html` powered by `assets/js/site.js`
- **Styles**: `assets/css/site.css` ŌĆö all site chrome styles; entries can have their own inline `<style>`

---

## Critical Rules

- **Never invent content.** Only add information explicitly provided by the user ŌĆö no descriptions, tags, directors, genres, etc. unless stated. Leave fields empty (`""`) if not provided.
- Always run `npm run build` after creating or modifying entries.
- When editing a file, always read it first (Edit tool requires a prior read).
- **Back links**: every entry's `ŌåÉ back` link must point to its collection page, not `/`:
  - movie ŌåÆ `/movies.html`, book ŌåÆ `/books.html`, theater ŌåÆ `/theater.html`
  - meal ŌåÆ `/meals.html`, outfit ŌåÆ `/outfits.html`, memory ŌåÆ `/memories.html`
  - travel ŌåÆ `/travel.html`, shelf ŌåÆ `/shelf.html`, interactive ŌåÆ `/interactive.html`
  - quote ŌåÆ `/quotes.html`, note ŌåÆ `/notes.html`, top5 ŌåÆ `/top5.html`

### top5
- **No individual entry files.** All top 5 lists live directly in `top5.html` as inline HTML ŌĆö do NOT create entries in `entries/`.

  ```html

    <div class="notebook-head"><h2>Title</h2></div>
    <div class="notebook-body">
      <div class="notebook-rule"></div>
      <ol class="notebook-list">
        <li>Item one</li>
        ...
      </ol>
    </div>
  </div>
  ```
- Pink rule: `border-bottom: 2.5px solid #e87878` on `.notebook-head`
- Ruled lines: `border-bottom: 1px solid #8cbde8` on each `li` and `.notebook-rule`; `margin: 0 -2rem; padding: 0 2rem` to extend full width
- One `.notebook-rule` blank line before the list (`height: 0.7rem`); `padding-bottom: 0.8rem` on `.notebook-body` after last item
- List items: `color: #1a1a5e` (dark navy), numbers via `::before` counter in `color: #bbb`
- Caveat font loaded in `top5.html` head; no `npm run build` needed when editing `top5.html`

## Photo Workflow

Photos for memories and meals are stored as **regular files** in the entry directory (not base64). This keeps `manifest.json` lean regardless of how many photo entries exist.

**Memory entries** ŌĆö two files per entry:
```bash
# Full size for entry page (photo.jpg):
sips -Z 900 -s format jpeg -s formatOptions 60 input.jpg --out photo.jpg

# Thumbnail for collection card (thumb.jpg):
sips -Z 400 -s format jpeg -s formatOptions 35 input.jpg --out thumb.jpg
```
- Save both in the entry directory (e.g. `entries/2026-02/0228-apres-ski/`)
- Metadata `"photo": "thumb.jpg"` ŌĆö just the filename, not a data URI
- Entry HTML: `<img src="photo.jpg">`

**Meal entries** ŌĆö one file per photo:
```bash
sips -Z 900 -s format jpeg -s formatOptions 60 input.jpg --out photo-1.jpg
# repeat for photo-2.jpg, photo-3.jpg etc.
```
- Save in the entry directory
- Entry HTML: `<img src="photo-1.jpg">`, `<img src="photo-2.jpg">` etc.
- No photo field in meal metadata (meals don't show photos in collection cards)

Target sizes after compression:
- Full / meal photo: ~100ŌĆō150KB
- Memory thumbnail: ~15ŌĆō25KB

**Exception**: If the user explicitly asks for base64 for a specific entry, embed as `data:image/jpeg;base64,...` in the HTML only ŌĆö never in metadata/manifest.

## Build & Deploy

```bash
npm run build        # regenerates manifest.json from all entries
```

**Deploy workflow:**
- After making changes, run `npm run build` and commit ŌĆö but **do NOT push**.
- Only run `git push` when the user explicitly says "deploy".
- Netlify auto-deploys on push to `main`.
